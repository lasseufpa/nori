46 /**
47  * \ingroup helper
48  * \brief Helper for a correct setup of every NR simulation
49  *
50  * This class will help you in setting up a single- or multi-cell scenario
51  * with NR. Most probably, you will interact with the NR module only through
52  * this class, so make sure everything that is written in the following is
53  * clear enough to start creating your own scenario.
54  *
55  * \section helper_pre Pre-requisite: Node creation and placement
56  *
57  * We assume that you read the ns-3 tutorials, and you're able to create
58  * your own node placement, as well as the mobility model that you prefer
59  * for your scenario. For simple cases, we provide a class that can help you
60  * in setting up a grid-like scenario. Please take a look at the
61  * GridScenarioHelper documentation in that case.
62  *
63  * \section helper_creating Creating the helper
64  *
65  * Usually, the helper is created on the heap, and have to live till the end
66  * of the simulation program:
67  *
68 \verbatim
69  Ptr<NrPointToPointEpcHelper> epcHelper = CreateObject<NrPointToPointEpcHelper> ();
70  Ptr<IdealBeamformingHelper> idealBeamformingHelper = CreateObject<IdealBeamformingHelper>();
71  Ptr<NrHelper> nrHelper = CreateObject<NrHelper> ();
72 
73  nrHelper->SetBeamformingHelper (idealBeamformingHelper);
74  nrHelper->SetEpcHelper (epcHelper);
75 \endverbatim
76  *
77  * As you can see, we have created two other object that can help this class:
78  * the IdealBeamformingHelper and the NrPointToPointEpcHelper. Please refer to
79  * the documentation of such classes if you need more information about them.
80  *
81  * \section helper_dividing Dividing the spectrum and creating the channels
82  *
83  * The spectrum management is delegated to the class CcBwpHelper. Please
84  * refer to its documentation to have more information. After having divided
85  * the spectrum in component carriers and bandwidth part, use the method
86  * InitializeOperationBand() to create the channels and other things that will
87  * be used by the channel modeling part.
88  *
89  * \section helper_configuring Configuring the cells
90  *
91  * After the spectrum part is ready, it is time to check how to configure the
92  * cells. The configuration is done in the most typical ns-3 way, through the
93  * Attributes. We assume that you already know what an attribute is; if not,
94  * please read the ns-3 documentation about objects and attributes.
95  *
96  *
97  * We have two different ways to configure the attributes of our objects
98  * (and, therefore, configure the cell). The first is before the object are
99  * created: as this class is responsible to create all the object that are needed
100  * in the module, we provide many methods (prefixed with "Set") that can
101  * store the attribute values for the objects, until they are created and then
102  * these values applied. For instance, we can set the antenna dimensions for
103  * all the UEs with:
104  *
105 \verbatim
106  // Antennas for all the UEs
107  nrHelper->SetUeAntennaAttribute ("NumRows", UintegerValue (2));
108  nrHelper->SetUeAntennaAttribute ("NumColumns", UintegerValue (4));
109 \endverbatim
110  *
111  * These attributes will be stored, and then used inside the creation methods
112  * InstallGnbDevice() and InstallUeDevice(). In this case, the antenna dimensions
113  * will be taken inside the InstallUeDevice() method, and all the UEs created
114  * will have these dimensions.
115  *
116  * Of course, it is possible to divide the nodes, and install them separately
117  * with different attributes. For example:
118  *
119 \verbatim
120  NodeContainer ueFirstSet;
121  NodeContainer ueSecondSet;
122  ...
123  nrHelper->SetUeAntennaAttribute ("NumRows", UintegerValue (2));
124  nrHelper->SetUeAntennaAttribute ("NumColumns", UintegerValue (4));
125  ...
126  NetDeviceContainer ueFirstSetNetDevs = nrHelper->InstallUeDevice (ueFirstSet, allBwps);
127  ...
128  // Then, prepare the second set:
129  nrHelper->SetUeAntennaAttribute ("NumRows", UintegerValue (4));
130  nrHelper->SetUeAntennaAttribute ("NumColumns", UintegerValue (8));
131  ...
132  NetDeviceContainer ueSecondSetNetDevs = nrHelper->InstallUeDevice (ueSecondSet, allBwps);
133  ...
134 \endverbatim
135  *
136  * In this way, you can configure different sets of nodes with different properties.
137  *
138  * The second configuration option is setting the attributes after the object are
139  * created. Once you have a pointer to the object, you can use the Object::SetAttribute()
140  * method on it. To get the pointer, use one of the helper methods to retrieve
141  * a pointer to the PHY, MAC, or scheduler, given the index of the bandwidth part:
142  *
143 \verbatim
144  ...
145  NetDeviceContainer enbNetDev = nrHelper->InstallGnbDevice (gnbContainer, allBwps);
146  nrHelper->GetGnbPhy (enbNetDev.Get (0), 0)->SetAttribute ("Numerology", UintegerValue (2));
147 \endverbatim
148  *
149  * In the snippet, we are selecting the first gnb (`enbNetDev.Get (0)`) and
150  * then selecting the first BWP (`, 0`). We are using the method GetGnbPhy()
151  * to obtain the PHY pointer, and then setting the `Numerology` attribute
152  * on it. The list of attributes is present in the class description, as well
153  * as some reminder on how configure it through the helper.
154  *
155  * \section helper_installing Installing UEs and GNBs
156  *
157  * The installation part is done through two methods: InstallGnbDevice()
158  * and InstallUeDevice(). Pass to these methods the container of the nodes,
159  * plus the spectrum configuration that comes from CcBwpHelper.
160  *
161  * \section helper_finishing Finishing the configuration
162  *
163  * After you finish the configuration, please remember to call UpdateConfig()
164  * on all the NetDevices:
165  *
166 \verbatim
167  for (auto it = enbNetDev.Begin (); it != enbNetDev.End (); ++it)
168  {
169  DynamicCast<NoriGnbNetdevice> (*it)->UpdateConfig ();
170  }
171 
172  for (auto it = ueNetDev.Begin (); it != ueNetDev.End (); ++it)
173  {
174  DynamicCast<NrUeNetDevice> (*it)->UpdateConfig ();
175  }
176 \endverbatim
177  *
178  * The call to UpdateConfig() will finish the configuration, and update the
179  * RRC layer.
180  *
181  * \section helper_bearers Dedicated bearers
182  *
183  * We have methods to open dedicated UE bearers: ActivateDedicatedEpsBearer()
184  * and DeActivateDedicatedEpsBearer(). Please take a look at their documentation
185  * for more information.
186  *
187  * \section helper_attachment Attachment of UEs to GNBs
188  *
189  * We provide two methods to attach a set of UE to a GNB: AttachToClosestEnb()
190  * and AttachToEnb(). Through these function, you will manually attach one or
191  * more UEs to a specified GNB.
192  *
193  * \section helper_Traces Traces
194  *
195  * We provide a method that enables the generation of files that include among
196  * others information related to the received packets at the gNB and UE side,
197  * the control messages transmitted and received from/at the gNB and UE side,
198  * the SINR, as well as RLC and PDCP statistics such as the packet size.
199  * Please refer to their documentation for more information.
200  * Enabling the traces is done by simply adding the method enableTraces() in the
201  * scenario.
202  *
203  */
